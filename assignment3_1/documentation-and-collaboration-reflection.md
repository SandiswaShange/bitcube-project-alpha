# Documentation and Collaboration Reflection

The original README assumed that readers already understood the purpose of the project and how the repository was organised. It contained very limited information and did not explain system functionality, onboarding requirements, or how project documentation was structured. This meant that a new contributor would likely struggle to understand where to begin.

The updated README improves onboarding by providing a project overview, system context, onboarding instructions, and explanations of key Sprint 1 markdown artefacts. It also explains the use of Pull Requests to introduce changes, helping new contributors understand project expectations.

Through this assignment, I learned that documentation and Pull Requests are important communication tools within a development team. A Pull Request helps explain why changes are made and what reviewers should focus on, while issue reporting structures improve clarity when identifying bugs or suggesting improvements. Proper documentation helps reduce confusion and supports collaboration within a software project.
